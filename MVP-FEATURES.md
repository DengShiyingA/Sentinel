# Sentinel MVP 功能边界详细说明

## 📋 MVP 核心目标

**用一句话概括**：让用户能通过 iOS 手机审批 Claude Code 的工具调用，并通过简单规则实现自动化。

## ✅ MVP 包含功能（按优先级排序）

### P0 - 必须有（核心价值）

#### 1. 基础审批流程 ⭐⭐⭐⭐⭐
- [ ] **CLI Hook 服务**
  - 启动 HTTP 服务器监听 Claude Code 的 PreToolUse hook
  - 接收工具调用信息（工具名、参数）
  - 30 秒超时机制，超时自动拒绝
  - 本地 SQLite 缓存（离线时的降级策略）

- [ ] **Server 转发与决策**
  - Socket.IO 连接管理（CLI 和 iOS 都连接到 server）
  - 接收 CLI 的工具调用请求
  - 查询规则引擎，决定是否需要审批
  - 推送审批请求到 iOS（Socket.IO 实时推送）
  - 等待 iOS 响应并返回给 CLI

- [ ] **iOS 审批界面**
  - 实时接收审批请求（Socket.IO）
  - 展示工具名称和参数（格式化 JSON）
  - 两个操作按钮：✅ 放行 / ❌ 拒绝
  - 返回决策到 server

- [ ] **端到端加密**
  - X25519 密钥对生成（CLI 和 iOS 各自生成）
  - NaCl 对称加密消息内容
  - 配对流程：
    1. CLI 生成二维码（包含 server URL + CLI 公钥）
    2. iOS 扫码，提取信息并上传 iOS 公钥
    3. Server 存储密钥映射，后续消息加密传输

#### 2. 基础规则引擎 ⭐⭐⭐⭐
- [ ] **规则类型（仅 3 种）**
  - **路径白名单**：匹配文件路径，自动放行
    - 示例：`/tmp/**` → allow
  - **工具黑名单**：禁止特定工具
    - 示例：`rm -rf` → block
  - **成本阈值**：单次调用超过金额需审批
    - 示例：`cost > 0.10` → require_approval

- [ ] **规则匹配逻辑**
  - 按优先级（priority 字段）从高到低匹配
  - 第一条匹配的规则决定行为
  - 无匹配规则时：默认 require_approval

- [ ] **规则管理（iOS）**
  - 列表展示所有规则
  - 创建规则（简单表单）
  - 启用/禁用规则（toggle）
  - 删除规则

#### 3. iOS 核心功能 ⭐⭐⭐⭐
- [ ] **配对流程**
  - 扫描 CLI 生成的二维码
  - 自动连接到 server
  - 保存配对信息到 Keychain

- [ ] **审批详情页**
  - 工具名称（大标题）
  - 参数展示（代码块，JSON 高亮）
  - 文件路径（如果有）
  - 操作按钮：放行 / 拒绝 / 本次允许 / 添加规则

- [ ] **Face ID 验证**
  - 高风险操作（riskLevel = 'high'）强制 Face ID
  - 使用 `LocalAuthentication` 框架
  - 失败自动拒绝

- [ ] **推送通知**
  - 后台收到审批请求时弹出通知
  - 点击通知跳转到审批详情页
  - APNs 配置（可选，无 APNs 时仅前台可用）

### P1 - 应该有（提升体验）

#### 4. 成本追踪 ⭐⭐⭐
- [ ] **记录工具调用成本**
  - CLI 上报 token 使用量和模型
  - Server 计算成本（简化公式，如 GPT-4 = $0.03/1K tokens）
  - 存储到 PostgreSQL

- [ ] **iOS 成本面板**
  - 今日/本周/本月总成本
  - 按工具分类统计（饼图或列表）
  - 简单时间趋势（折线图，用 SwiftUI Charts）

- [ ] **预算告警**
  - 设置每日/每周预算
  - 超过预算时弹出通知
  - 审批界面显示"已超预算"警告

#### 5. 审批上下文增强 ⭐⭐⭐
- [ ] **文件 Diff 展示**
  - 如果工具是 `write_file`，显示文件内容差异
  - 简单的语法高亮（用 `highlightjs` 或纯文本）
  - iOS 上以代码块形式展示

- [ ] **风险评估**
  - Server 端简单规则判断风险等级：
    - `rm`、`sudo`、`chmod` → high
    - `write_file`（重要路径）→ medium
    - 其他 → low
  - iOS 界面用颜色区分（红/橙/绿）

#### 6. 基础设施 ⭐⭐⭐⭐
- [ ] **Docker Compose 部署**
  - `docker-compose up -d` 一键启动
  - PostgreSQL + Redis + Sentinel Server
  - 包含健康检查

- [ ] **环境变量配置**
  - `.env` 文件管理所有配置
  - 示例配置文件 `.env.example`

- [ ] **数据库迁移**
  - Prisma 迁移脚本
  - 启动时自动运行 `prisma migrate deploy`

- [ ] **健康检查接口**
  - `GET /health` 返回服务状态
  - 检查数据库和 Redis 连接

### P2 - 可以有（锦上添花）

#### 7. 审批历史 ⭐⭐
- [ ] **iOS 历史列表**
  - 显示所有已处理的审批请求
  - 筛选：全部/已批准/已拒绝
  - 支持搜索（按工具名）

- [ ] **详情回放**
  - 点击历史记录查看详情
  - 不可编辑，仅查看

#### 8. CLI 体验优化 ⭐⭐
- [ ] **彩色日志**
  - 用 `chalk` 输出彩色日志
  - 区分 INFO / WARN / ERROR

- [ ] **配对状态检查**
  - `sentinel status` 命令查看连接状态
  - 显示已连接设备列表

- [ ] **本地缓存**
  - SQLite 缓存审批决策（离线时降级）
  - 重新连接后同步到 server

#### 9. 规则模板 ⭐
- [ ] **预设规则模板**
  - 新用户首次登录时自动创建 3 条规则：
    1. `/tmp/**` → allow
    2. `rm -rf` → block
    3. `cost > 0.10` → require_approval

- [ ] **快速添加规则**
  - 审批界面点击"添加规则"
  - 自动填充当前工具/路径信息
  - 一键保存

## 📊 MVP 验收标准

### 用户故事 1：首次使用
1. ✅ 用户在 VPS 上运行 `docker-compose up -d`，服务启动成功
2. ✅ 在开发机上安装 `sentinel-cli`，运行 `sentinel pair`
3. ✅ 在 iOS 设备上打开 Sentinel 应用，扫描二维码完成配对
4. ✅ 配置 Claude Code 的 hook 指向 CLI
5. ✅ Claude Code 执行工具时，iOS 收到审批请求并弹出通知
6. ✅ 用户在 iOS 上点击"放行"，Claude Code 继续执行

### 用户故事 2：创建规则
1. ✅ 用户在 iOS 应用中进入"规则"页面
2. ✅ 点击"+"创建规则，选择"路径白名单"
3. ✅ 输入 `/tmp/**`，动作选择"自动放行"
4. ✅ 保存规则
5. ✅ Claude Code 再次操作 `/tmp/` 路径时，自动放行，不再弹出审批

### 用户故事 3：成本追踪
1. ✅ 用户执行多个工具调用
2. ✅ 在 iOS 的"成本"页面看到今日总成本
3. ✅ 查看按工具分类的成本统计
4. ✅ 设置每日预算 $1.00，超过后收到告警通知

## 📝 技术债务记录

MVP 中的权衡决策：

1. **规则引擎简化**：仅支持 3 种规则类型，未来可扩展为 DSL
2. **无 Web 后台**：仅 iOS 管理，降低复杂度
3. **单用户模式**：一个 server 对应一个用户，团队功能后续再做
4. **简单成本计算**：使用固定费率，不考虑实时价格波动
5. **无离线模式**：CLI 必须连接到 server，SQLite 仅作缓存
